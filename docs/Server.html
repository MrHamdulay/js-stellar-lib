accounts(resourceopt, optsopt) → {Promise|EventSource}

Returns account resources. For a list of all accounts, don't pass an address or resource parameters. For a specific transaction, pass only an address. For a list of account sub resources, pass the account address and the type of sub resources.

A configuration object can be passed to each call. Calls that return a collection can be streamed by passing a streaming object in the config object.

@param {string} [address] - Returns the given account.
Parameters:
Name Type Attributes Description
resource string <optional>
 Return a specific resource associated with an account. Can be {"transactions", "operations", "effects"}.
opts object <optional>
 Optional configuration for the request.
Properties
Name Type Attributes Description
after string <optional>
 Return only resources after the given paging token.
limit number <optional>
 Limit the number of returned resources to the given amount.
order string <optional>
 Order the returned collection in "asc" or "desc" order.
streaming object <optional>
 Streaming handlers. If not null, will turn this request into a streaming request.
Properties
Name Type Attributes Description
onmessage function <optional>
 If given an onmessage handler, turns this query into a streaming query.
onerror function <optional>
 If this query is a streaming query, will use the given handler for error messages.

If this is a normal request (non streaming) will return a promise that will fulfill when the request completes. Otherwise, it will return the EventSource object.
